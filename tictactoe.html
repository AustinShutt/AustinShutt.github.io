SFML Tic-Tac-Toe


This application is built using C++ and the SFML library.

The source code for this project is available on its public github repository.

Design Requirements

Purpose:
This was a project for a client on my Fiverr page that requested an SFML application that implemented the minimax algorithm for the computer player. All the used assets were created by me in Aseprite. (Yes I know it's just X's and O's).

Requirements:

  1. SFML staticly linked library for project.
  2. Minimax algorithm usage for computer opponent.

Main Game Loop

Below is a flowchart of the main game-loop of the entire application. This is a simple if inefficient approach where the menu state stays in the call stack of the game stack. But for such a small game this is an appropriate method to switch back to the main menu state when the user decides to exit the Tic-Tac-Toe game and return to the menu.

Game Logic

Below is a flowchart for all possible actions that can be taken while inside of the game state. The crux of the algorithm takes place in the decision diamond with "CurrentPtr == Human Player?", if the instance is not a human player then the AI makes a move by overriding the virtual function inside the player class. This allows for an easy flow of information through the application and a logic pattern that is easy to reason about while using OOP design principles.
